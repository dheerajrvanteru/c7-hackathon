Agent Design

Each agent's role, tools, inputs, and outputs in the shared SecurityState

📋 Log Monitor Agent
TOOLS
parse_logs(source)
detect_anomalies(entries)
classify_severity(event)
DETECTS
SSH brute force, port scans
Failed auth spikes
Unusual traffic patterns
OUTPUT → STATE
anomalies[], severity_map{}
🔍 Threat Intelligence Agent
TOOLS
query_nvd_cve(keyword)
lookup_ip_reputation(ip)
match_attack_patterns(anomaly)
DATA SOURCES
NVD API (real CVE data)
MITRE ATT&CK patterns
AbuseIPDB for IP lookups
OUTPUT → STATE
cve_matches[], threat_score
🛡️ Vulnerability Scanner Agent
TOOLS
scan_code_snippet(code)
check_api_headers(url)
audit_docker_image(name)
CHECKS
OWASP Top 10 patterns
Missing security headers
Outdated base images
OUTPUT → STATE
vulnerabilities[], risk_level
🚨 Incident Response Agent
TOOLS
generate_action_plan(findings)
prioritize_threats(list)
draft_runbook(incident)
READS FROM STATE
anomalies, cve_matches
vulnerabilities, threat_score
OUTPUT → STATE
action_plan[], runbook_md
📜 Policy Checker Agent
TOOLS
check_nist_compliance(findings)
check_soc2_controls(findings)
generate_compliance_report()
STANDARDS
NIST CSF 2.0
SOC 2 Type II controls
ISO 27001 basics
OUTPUT → STATE
compliance_gaps[], score%
